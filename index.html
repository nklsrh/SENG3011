  • What's new in this release?

    Implementation

    Input Selects trade transactions from a list of transactions in the inputFile.csv file, then calculates its return over time.

    Arguments from the arguments.txt file (sample) are now utilised and must be specified, each on its own line.

    (Line 1) Argument 1: Trade Window This is used to calculate the number of returns to be included in the calculation of the Simple Moving Average.

    (Line 2) Argument 2: Threshold This dictates the values at which to generate specific trading signals (buy/sell) based on the Simple Moving Average.

    Output A list of approved actions are saved to a new file called OrderList.csv (sample). This indicates what equities should be bought or sold.

    The functions are logged and saved to a file called MomentumStrategyModule.log, a log report containing details about the file (number of TRADES, LINES, execution time), which is saved to the working directory.

    Updates

    NEW: MSM_v2 now takes arguments specified in the arguments.txt file. These arguments are 'window' and 'threshold'.

    The arguments.txt file CANNOT be left blank!

    NEW: Performance optimisation resulting in a smaller footprint and memory usage. This was accomplished with extensive code refactoring and moving from a LinkedList-based system to an ArrayList system.

To run
  • 1. Open cmd and go to the downloaded JAR's directory
  • 2. Type the following:

    java -jar MSM_v2.jar inputFile.csv arguments.txt

    Where
    java -jar is the function to run a JAR file
    MSM_v2.jar is the program module to be run
    inputFile.csv is the input file
    arguments.txt is a text file which includes parameters needed for the module

  • 3. A log saved to a file MomentumStrategyModule.log
  • 4. A list of approved actions are saved to a new file OrderList.csv. This indicates what equities should be bought or sold.
